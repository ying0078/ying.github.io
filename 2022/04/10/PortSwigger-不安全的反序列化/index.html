PortSwigger-不安全的反序列化

0x01 前言

  • 刷这个专题的时候是有一点反序列化漏洞的基础了，主要是 PHP 的反序列化，魔法函数的使用。所以对序列化已经有一点基础的了解了。

0x02 概念明晰

先弄懂序列化，反序列化，不安全的反序列化都是什么

1.序列化是什么，反序列化是什么

  • 序列化，将复杂的东西转化为编码的操作，比如将类转化成一串编码，实现方式是 .serialization()

以这串 php 代码为例，体验一下序列化

$user->name = "carlos"; 
$user->isLoggedIn = true;

输出—> O:4:"User":2:{s:4:"name":s:6:"carlos"; s:10:"isLoggedIn":b:1;}

O:4:"User" - 具有 4 个字符的类名的对象 "User"
2 - 对象有2个属性
s:4:"name" - 第一个属性的键是 4 个字符的字符串 "name"
s:6:"carlos" - 第一个属性的值是 6 个字符的字符串 "carlos"
s:10:"isLoggedIn" - 第二个属性的键是 10 个字符的字符串 "isLoggedIn"
b:1 - 第二个属性的值是布尔值 true
  • 反序列化，将编码转换为复杂的东西，比如将编码转换为一个类，实现方式是 .deserialization

以这串 php 代码为例，体验一下反序列化

 $str = 'O%3A4%3A%22test%22%3A2%3A%7Bs%3A10%3A%22%00test%00flag%22%3Bs%3A9%3A%22flag%7B233%7D%22%3Bs%3A1%3A%22a%22%3Bs%3A3%3A%22aaa%22%3B%7D';

 $data = urldecode($str);
 $obj = unserialize($data);
 var_dump($obj);
 ?>
     
输出---> O:4:"test":2:{s:10:"testflag";s:9:"flag{233}";s:1:"a";s:3:"aaa";}

序列化与反序列化之间可以互相转换，也就是说，我序列化操作过去的东西，一定能够通过反序列化原模原样得拿回来，原理如图所示。

2. 什么是不安全的反序列化

不安全的反序列化是指进行序列化与反序列化操作的数据可以被用户控制，从而使得用户得以操作序列化对象，造成各种攻击。

不安全的反序列化主要被分为两类：

序列化对象被替换

这种不安全的反序列化，应用最多的便是 Public 的对象被修改为 Private，轻则替换类的范围(也就是 Public，Private 之间的)；重则替换一整个类，将序列化对象替换为完全不同类的对象。因此，不安全的反序列化有时称为 “对象注入” 漏洞。

在反序列化之前发生异常，导致反序列化直接被跳出

  • 最有代表性的可以是 php 反序列化中的 __destruct 魔法函数，师傅们可以移步至 php反序列化 观看

这种情况在 Port 官方靶场上被称作意外类的对象导致的异常。

3. 产生不安全的反序列化的根本原因

一句话总结：过于信任用户的输入

4. 不安全的反序列化的危害

  • 最有可能的是 RCE，我们熟知的 Shiro 550，Shiro 721 以及其他的一些反序列化漏洞。
  • 其他次要一些的有提权、任意文件读取、DDoS、信息泄露

0x03 开打！不安全反序列化的利用

1. 判断不安全的反序列化的存在

什么叫判断漏洞存在

  • 判断漏洞存在的意思比较好理解，也就是证明当前界面存在某个漏洞。例如我们要证明某个 Web 中存在 XSS，我们只需要构造 alert(1) 的 POC 即可。对于不安全的反序列化也是一样。

    如何识别不安全的反序列化漏洞

    识别不安全的反序列化比较简单，不论是在黑盒测试还是白盒测试当中。

在渗透测试的过程中，抓包查看数据，审计看起来像序列化是数据，例如 token，session 的地方。对这些地方进行测试，观察能否控制输入。

2. 修改数据类型

修改数据类型多数出现在 PHP 弱判断的题目当中。

php中有两种比较的符号

  • 它们分别是 == 与 ===
1 <?php
2 $a = $b ;
3 $a===$b ;
4 ?>

=== 在进行比较的时候，先判断两种字符串的类型是否相等，再比较；三个等于号的判断方式被称作严格判断

== 在进行比较的时候，会先将字符串类型转化成相同，再比较。也被称作松散比较

// php的"=="
1 <?php
2 var_dump("admin"==0);  //true
3 var_dump("1admin"==1); //true
4 var_dump("admin1"==1) //false
5 var_dump("admin1"==0) //true
6 var_dump("0e123456"=="0e4456789"); //true 
7 ?>  //上述代码可自行测试

当这种松散比较与过于信任用户输入同时出现时，肯定有洞。

例如下面这一段逻辑判断代码，只要存储的密码不以数字开头，那么松散比较的返回结果始终为 True

$login = unserialize($_COOKIE) 
if ($login['password'] == $password) { 
    // log in successfully 
}

这时的你是否已经有攻击思路了呢，我们以 Port 靶场体验一下。

4. 利用魔术方法进行反序列化攻击

  • 魔术方法的定义，魔术方法是方法的特殊子集，一般情况下不会出现的，但每当发生特定事件或方案时，都会被自动调用。

我帮大家理出一下 PHP，Java，Python 都有哪些魔法函数

__construct()    #类的构造函数  
__destruct()    #类的析构函数  
__call()    #在对象中调用一个不可访问方法时调用  
__callStatic()    #用静态方式中调用一个不可访问方法时调用  
__get()    #获得一个类的成员变量时调用  
__set()    #设置一个类的成员变量时调用  
__isset()    #当对不可访问属性调用isset()或empty()时调用  
__unset()    #当对不可访问属性调用unset()时被调用。  
__sleep()    #执行serialize()时，先会调用这个函数  
__wakeup()    #执行unserialize()时，先会调用这个函数  
__toString()    #类被当成字符串时的回应方法  
__invoke()    #调用函数的方式调用一个对象时的回应方法  
__set_state()    #调用var_export()导出类时，此静态方法会被调用。  
__clone()    #当对象复制完成时调用  
__autoload()    #尝试加载未定义的类  
__debugInfo()    #打印所需调试信息

不同的魔法函数都可以构造对应的不安全的反序列化，但是攻击者构造 POC 的前提是获得到了后端判断的源代码，进行代码审计，从而设计 POC，否则将难以攻击。

  • 下面是一道简单的 PHP 魔法函数的代码审计。

  • 尝试 /libs/CustomTemplate.php~，在 PHP 中，结尾带有 ~，很可能是备份文件。访问成功，把源码拉下来。

在后面看到有 PHP 的魔法函数 ———— __construct() 与 __destruct()，感觉可以利用。毕竟是代码审计嘛，我们把两个魔法函数拉出来审计一下。

  • __construct 函数进行了简单赋值的工作
public function __construct($template_file_path) 
{
 $this->template_file_path = $template_file_path;
 $this->lock_file_path = $template_file_path . ".lock";
 }
  • __destruct() 中的 unlink 函数用于删除文件，这里给我们提供了攻击思路，只需要我们修改 lock_file_path 为 /home/carlos/morale.txt 文件即可进行攻击。
function __destruct() 
{
 // Carlos thought this would be a good idea
 if (file_exists($this->lock_file_path)) 
 {
 unlink($this->lock_file_path);
 }
 }

先使用 Port 提供给我们的 wiener 与 peter 登录，再对 Accout 界面进行抓包，将 lock_file_path 修改为 /home/carlos/morale.txt

将 Session 值放入 Decoder 模块，并修改为如下

O:14:"CustomTemplate":1:{s:14:"lock_file_path";s:23:"/home/carlos/morale.txt";}

将我们编写好的 payload 进行编码，替换 Session；发包即可。

5. Gadget Chain 一些反序列化构造的工具

  • 这一块主要是 Gadget Chain 这一概念，之前自己学习的时候也是概念不清楚，卡了一会儿。

Gadget Chain 其实就是我们所说的通用 payload，在我们输入要任意执行的命令 ———— 例如 whoami,ipconfig 等命令时，Gadget Chain 会帮我们进行自动编码，减轻负担。

Java反序列化的一个这样的工具是 “ysoserial”。我们在使用它的时候先选择对应的框架，然后传入要执行的命令。

然后，它基于所选链创建适当的序列化对象，当然有一定的精准度，但是可以提高我们的渗透效率。

  • 我个人把它理解为黑盒测试，不断尝试 payload。
  • 这里 Port 向我们介绍了一样工具 ———— ysoserial，用来进行反序列化的构造。
